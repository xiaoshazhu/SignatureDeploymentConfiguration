# 技术选择及要求标准 (前端)

## 🛠 技术栈
| 模块 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| **核心框架** | **Vue 3** | 采用 Composition API，确保代码逻辑复用性与清晰度。 |
| **UI 组件库** | **Element Plus** | 用于构建配置面板 (`ConfigPanel`)，提供标准的表单、按钮与弹窗组件。 |
| **插件 SDK** | **@lark-base-open/js-sdk** | 仅用于获取多维表格上下文 (TableId, RecordId) 和触发简单的界面交互。 |
| **HTTP 客户端** | **Fetch API / Axios** | 用于与 Python 后端进行数据交互 (生成链接、提交签名)。 |
| **签字组件** | **vue-signature-pad** | 轻量级 Canvas 签名组件，支持 PC 与移动端手写输入。 |

## 📏 开发标准

### 1. 架构原则：Backend-Centric
*   **只读与触发**：前端原则上**不直接修改**多维表格的字段值（除了开发阶段的调试）。
*   **API 优先**：所有涉及数据变更的操作（如批量生成链接、更新状态、上传图片），必须调用后端 API 执行。
*   **Token 安全**：严禁在前端代码中硬编码 `app_secret` 或 `tenant_access_token`。

### 2. 组件职责
*   **ConfigPanel.vue (配置面板)**：
    *   负责收集用户配置（模式、人数、开关）。
    *   调用后端 `/api/batch/generate-links` 接口触发批量处理。
    *   监听并展示后端返回的操作日志。
*   **SignPage.vue (H5 签字页)**：
    *   负责展示行详情（手风琴组件）。
    *   负责捕获用户签名（Canvas）。
    *   负责调用后端 `/api/sign/submit` 提交数据。
    *   **响应式设计**：必须同时适配移动端竖屏（宽度 100%）和 PC 端浏览器。

### 3. 交互体验
*   **加载状态**：所有异步请求（API 调用）期间，按钮必须进入 Loading 状态，防止重复点击。
*   **错误处理**：API 返回非 200 状态码或 `code != 0` 时，必须通过 Toast 或 Alert 提示具体的 `msg` 错误信息。
*   **防抖保护**：签字提交成功后，应立即禁用提交按钮，并显示成功状态，防止用户重复提交。

### 4. 代码规范
*   **命名**：组件名使用大驼峰 (PascalCase)，如 `ConfigPanel.vue`；变量与方法使用小驼峰 (camelCase)。
*   **注释**：关键业务逻辑（如 API 调用参数组装、状态判断）必须包含简体中文注释。

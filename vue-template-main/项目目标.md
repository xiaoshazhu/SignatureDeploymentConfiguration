# 前端项目目标 (vue-template-main)

## 1. 项目定位
本项目是飞书多维表格电子签名系统的**前端部分**，主要承担“用户交互界面”和“业务触发器”的角色。它不直接处理复杂的数据写入逻辑，而是通过调用后端 API 来实现业务功能。

## 2. 核心模块

### 2.1 配置面板 (ConfigPanel)
**运行环境**：飞书桌面端多维表格插件侧边栏。
**主要功能**：
1.  **参数配置**：
    *   **签字模式**：支持“会签”（AND）和“或签”（OR）。
    *   **签字人数**：设定需要签字的总人数。
    *   **高级选项**：开启/关闭“生成签字二维码”功能。
2.  **操作触发**：
    *   **批量生成链接**：点击按钮，调用后端 `/api/batch/generate-links` 接口，对全表数据生成签字链接。
    *   **单行生成链接**：点击按钮，弹出配置框，调用后端接口为特定行生成定制化的签字链接。
3.  **状态反馈**：
    *   实时显示后端 API 的处理进度和结果（成功/跳过/失败）。
    *   展示操作日志，方便排查问题。

### 2.2 H5 签字页 (SignPage)
**运行环境**：移动端（飞书/微信）或 PC 端浏览器。
**主要功能**：
1.  **信息展示 (手风琴)**：
    *   页面顶部默认折叠展示当前行的详细数据（Key-Value）。
    *   用户点击可展开查看，确保“所签即所见”，防止误操作。
2.  **签字画板 (Canvas)**：
    *   提供全屏或大区域的签字画板。
    *   支持撤销、重写功能。
    *   **自适应布局**：完美适配手机竖屏和 PC 宽屏。
3.  **历史签名复用**：
    *   初始化时查询后端，若当前用户有历史签名，显示“一键复用”按钮。
    *   点击复用后，直接提交历史签名的 Token，无需重新绘制。
4.  **防重复提交**：
    *   页面加载时检查状态，若已完成签字，则直接展示“已完成”状态和最终签名图，禁用提交按钮。

## 3. 技术要求
*   **框架**：Vue 3 + Element Plus (配置页) / Vant (H5页，可选)。
*   **通信**：
    *   与飞书交互：使用 `@lark-base-open/js-sdk` 获取 `tableId`, `viewId`, `recordId` 等上下文信息。
    *   与后端交互：使用 `fetch` 或 `axios` 调用 Python 后端 API。
*   **安全性**：
    *   不存储 `app_secret` 等敏感信息。
    *   所有写操作通过后端代理，前端仅负责读取上下文和展示。

## 4. 目录结构说明
*   `src/components/ConfigPanel.vue`: 插件配置主界面。
*   `src/components/SignPage.vue`: H5 签字页面。
*   `src/utils/bitableSetup.js`: 飞书 SDK 辅助工具（仅用于读取信息和触发简单操作）。

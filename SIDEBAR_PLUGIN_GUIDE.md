# 飞书多维表格边栏插件开发规范与指南

本文档总结了飞书多维表格（Bitable）边栏插件开发的官方标准、关键技术点及注意事项，作为本项目功能扩展（如企业计费、配额管理）的指导规范。

## 1. 核心上下文获取 (Context Recognition)

为了实现按企业计费，必须准确识别当前的租户（企业）信息。应使用 `@lark-base-open/js-sdk` 提供的以下方法：

- **获取企业唯一标识**：`bitable.bridge.getTenantKey()`
  - **用途**：作为计费系统的 Primary Key，用于绑定企业配额。
  - **返回值**：`Promise<string>`
- **获取用户唯一标识**：`bitable.bridge.getUserId()`
  - **用途**：用于记录具体操作人，或实现个人级别的免费试用限制。
  - **返回值**：`Promise<string>`

## 2. 鉴权架构标准 (Authentication)

- **前端 (User Context)**：插件在飞书客户端内运行，自带用户登录态。调用 JS SDK 无需手动传递 Token。
- **后端 (Service Context)**：
  - **推荐方案**：使用 **`PersonalBaseToken`** (个人授权码)。
  - **获取方式**：用户在多维表格界面手动生成。
  - **优势**：配置简单，权限由生成该 Token 的用户决定，非常适合 Bitable 插件的后端集成。
  - **注意**：后端调用飞书 API（如 `baseopensdk`）时，必须在 Header 中携带此 Token。

## 3. 安全与网络要求 (Security)

- **全量 HTTPS**：所有前端资源、静态资源及后端接口 **必须** 使用 HTTPS 协议。
- **域名白名单**：虽然边栏插件对域名限制较宽，但在高安全等级的企业租户中，建议确保服务域名稳定且不被企业防火墙拦截。
- **跨域 (CORS)**：后端必须允许来自 `*.feishu.cn` 的跨域请求。

## 4. UI 交互规范 (User Experience)

- **侧边栏适配**：边栏宽度有限（通常为 300px-400px）。
  - 使用响应式布局（Flex/Grid）。
  - 充值套餐、价格表等复杂信息建议使用 `el-dialog` 弹窗展示，避免在窄小的边栏挤占空间。
- **视觉一致性**：推荐使用 Element Plus 或飞书 Design System，保持按钮、字体与飞书原生体验一致。

## 5. 开发注意事项 (Gotchas)

- **Iframe 限制**：插件运行在 Iframe 中，注意处理某些被限制的浏览器 API。
- **租户隔离**：在开发计费系统时，务必在数据库层面实现严格的 `tenant_key` 隔离，确保 A 企业的配额不会被 B 企业误用。
- **双端兼容**：虽然主要是 PC 端边栏，但考虑到部分签字场景可能涉及移动端，生成的链接必须支持公网访问及移动端渲染。

## 6. 计费系统实施标准 (Billing Standard)

- **初始化**：新企业（Tenant）首次打开插件时，后端自动插入 `tenant_key` 并授予 30 次免费额度。
- **扣费点**：批量生成链接成功、或单次生成链接成功时。
- **错误排查**：若由于权限（Forbidden）导致生成失败，不应计入划算扣费。
